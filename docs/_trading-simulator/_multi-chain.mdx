## Supported blockchains

The Trading Simulator supports two main blockchain types:

### EVM chains

Ethereum Virtual Machine (EVM) compatible chains include:

- **Ethereum (eth)**: The original EVM blockchain
- **Polygon (polygon)**: Ethereum scaling solution
- **Base (base)**: Coinbase's L2 scaling solution
- **Optimism (optimism)**: L2 scaling solution
- **Arbitrum (arbitrum)**: L2 scaling solution
- **Avalanche (avalanche)**: High throughput EVM chain
- **BSC (bsc)**: Binance Smart Chain
- **Linea (linea)**: Consensys L2 scaling solution

### SVM chains

Solana Virtual Machine (SVM) chains include:

- **Solana (svm)**: High-performance blockchain

<Callout type="info">
  Each chain has its own token ecosystem, gas fee mechanism, and token pricing.
</Callout>

## Chain identifiers

When working with the Trading Simulator API, you'll need to use chain identifiers in two formats:

1. **Chain Type**: The blockchain type (`evm` or `svm`)
2. **Specific Chain**: The specific blockchain network (`eth`, `polygon`, `svm`, etc.)

For example, when executing a cross-chain trade:

```javascript
const crossChainTrade = {
  fromToken: "0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2", // WETH on Ethereum
  toToken: "0x7D1AfA7B718fb893dB30A3aBc0Cfc608AaCfeBB0", // MATIC on Polygon
  amount: "0.5",
  fromChain: "evm",
  toChain: "evm",
  fromSpecificChain: "eth",
  toSpecificChain: "polygon",
};
```

## Chain-specific tokens

Each blockchain has its native token and a set of supported tokens:

- **Native Tokens**: ETH for Ethereum, MATIC for Polygon, SOL for Solana, etc.
- **Stablecoins**: USDC, USDT, DAI available on most chains
- **Cross-Chain Tokens**: Many tokens are available on multiple chains (like USDC)

To get tokens available on a specific chain:

```javascript
// Get all tokens on Polygon
const response = await tradingClient.get("/tokens?chain=evm&specificChain=polygon");
const polygonTokens = response.data.tokens;
```

## Trading modes

The Trading Simulator supports two trading modes:

### Same-chain trading

Trading tokens within the same blockchain:

```javascript
// Trade ETH for USDC on Ethereum
const sameChainTrade = {
  fromToken: "0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2", // WETH
  toToken: "0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48", // USDC
  amount: "0.5",
};

const response = await tradingClient.post("/trade", sameChainTrade);
```

### Cross-chain trading

Trading tokens across different blockchains:

```javascript
// Trade SOL for ETH (Solana to Ethereum)
const crossChainTrade = {
  fromToken: "So11111111111111111111111111111111111111112", // SOL
  toToken: "0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2", // WETH
  amount: "5.0",
  fromChain: "svm",
  toChain: "evm",
  fromSpecificChain: "svm",
  toSpecificChain: "eth",
};

const response = await tradingClient.post("/trade", crossChainTrade);
```

<Callout type="warning">
  Cross-chain trading may be disabled in some competitions. Check the competition rules and the
  `allowCrossChainTrading` setting in the API response.
</Callout>

## Trading fees and slippage

Each chain has its own fee structure and slippage characteristics:

- **EVM Chains**: Gas fees vary by chain (Ethereum typically has higher fees than Polygon)
- **Solana**: Lower fees but different token availability
- **Cross-Chain Trades**: May incur additional fees to simulate bridge costs

## Multi-chain portfolio management

### Viewing chain-specific balances

To view your balances by chain:

```javascript
const response = await tradingClient.get("/portfolio");
const portfolio = response.data.portfolio;

// Get Ethereum balances
const ethBalances = portfolio.chains.eth;
console.log("Ethereum total value:", ethBalances.totalValue);

// Get Solana balances
const solBalances = portfolio.chains.svm;
console.log("Solana total value:", solBalances.totalValue);
```

### Portfolio diversification strategies

Consider these strategies for multi-chain portfolios:

1. **Chain diversification**: Spread assets across multiple chains to reduce chain-specific risks
2. **Asset diversification**: Hold different types of assets on each chain
3. **Correlation management**: Choose assets with low correlation across chains
4. **Risk-adjusted allocation**: Allocate more to chains with better risk-reward profiles

## Performance analysis

The simulator provides performance metrics across all chains:

```javascript
const response = await tradingClient.get("/team/performance");
const performance = response.data.performance;

// Overall metrics
console.log("Total return:", performance.totalReturn);
console.log("Sharpe ratio:", performance.sharpeRatio);

// Chain-specific metrics
for (const chain in performance.chains) {
  console.log(`${chain} return:`, performance.chains[chain].return);
}
```
