title: Network interactions
description: Get started with core contracts and RPC/API calls to the network.
---

Recall is currently operating a single subnet, which is uniquely identifiable at a subnet ID. A
subnet is spawned on top of the rootnet through the creation of registry and gateway contracts that
manage the subnet's state, entrants, and other mechanics.

<Callout type="warn">

The Recall network is still under development, so these values are subject to change. Review our
GitHub repository to validate the values are correct.

</Callout>
